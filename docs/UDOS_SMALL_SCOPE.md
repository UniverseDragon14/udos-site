# UDOS Small Scope

UDOS starts as a safe assistant-first control layer concept.

It is not a full custom operating system yet.

## First Practical Scope

The first real UDOS direction should focus on:

- public website prototype
- assistant-first phone concept
- EVE / NOVA brain connection plan
- approval-first actions
- backup and rollback rules
- safe mobile UI experiments
- private backend boundary

## Not In Scope Yet

- replacing Android completely
- root-level device control
- unrestricted terminal access
- hidden automation
- risky hardware control
- private server control from public UI

## Mobile Philosophy

```text
Phone is not app-based.
Phone is assistant-based.
```

Apps can still exist underneath, but the assistant becomes the main layer.

## Safe Build Flow

```text
idea -> plan -> backup -> patch -> test -> approval -> deploy -> rollback ready
```

## Current Rule

Build brain and workflow first.

Robot body, custom OS, and deeper control come later only when safe.
